# 功能需求差距与 HTML 原型 UI 改造任务清单

## 目标

对照 `docs/功能需求.docx` 和 `docs/toy-td-ui-prototype.html`，整理当前项目还未完成或仍偏原型的内容，并把关卡挑战链路的 HTML 原型改造任务合并到同一份执行清单中。

本阶段优先改造关卡挑战相关 UI：选关界面、关卡侦察、战前配置。研究所、仓库和战斗 HUD 的正式化需求先记录在本文档中，后续再拆实现任务。

## 本轮实现状态

- 已完成 P0 关卡挑战链路改造：`UTTDBattleLevelSelectWidget` 改为顶部导航、左侧章节地图、右侧 `关卡侦察 / 战前配置` 双状态面板。
- 已完成战前配置同类玩具盒重复选择：选择会减少本次临时库存，返回侦察会恢复临时选择，开始战斗时聚合为 `FTTDNameStack`。
- 已扩展 `TTDUIStyle` 和按钮变体，沉淀原型色彩、关卡节点、卡片、槽位、幽灵按钮等样式入口。
- 已补研究所图鉴详情弹窗，玩具盒详情中的零件列表使用横向滚动。
- 已将仓库整理为卡片式库存页。
- 已将 Battle HUD 玩具盒购买 / 打开整理为固定商品商店卡片，并根据金币和库存刷新按钮状态。
- 已将 `测试补给` 按钮改为 `UTTDDeveloperSettings::bShowBattleTestCheatButton` 控制，默认隐藏。
- 已补战斗胜利覆盖层，显示返回选关倒计时。
- 已补敌人金币概率掉落字段 `CurrencyDropChance`，并补自动化测试覆盖 0 和 1 的边界。
- 已补建筑运行属性 `MoveSpeed`，默认 `0` 表示首版建筑不移动，并支持高度效果修饰。
- Editor build 的 C++ 编译阶段已通过；当前机器有正在运行的 `UnrealEditor.exe` 占用 `UnrealEditor-ToyTowerDefense.dll`，导致链接和自动化测试需要关闭 Editor 后再跑。

## 功能需求差距总览

### 未完成 / 明显不符合

- UI 没按需求图和 HTML 原型实现：开始界面、主界面、研究所、选关、准备窗口目前仍是 C++ UMG 原型，缺少图片式按钮、地图节点、弹窗和卡片视觉。
- 图鉴详情不是弹出说明窗口，当前是下方或面板文本反馈。
- 玩具盒说明里的零件列表左右滑动未实现，当前只是文本列出零件。
- 选关界面不是点击地图关卡位置弹出准备窗口，当前是纵向关卡列表。
- 关卡准备窗口中图纸和玩具盒是文本按钮，不是图片卡片。
- 玩具盒选择不支持同类多选数量变化，当前 `SelectedToyBoxIds` 是 `TSet<FName>`，同类只能选一次。
- 敌人死亡金币掉落不是概率，当前是固定 `CurrencyDrop`。
- 建筑属性里的移动速度未建模，当前建筑运行属性没有 `MoveSpeed`。
- 商店只是 Battle HUD 右侧购买 / 打开按钮，不是正式固定商品商店界面。
- 战斗胜利不是弹窗式提示，当前只在 HUD 状态中显示胜利并在延迟后返回选关。
- Battle HUD 仍有 `测试补给` 调试按钮，正式流程中应隐藏或受开发开关控制。

### 部分完成但仍偏原型

- 研究所研发页签核心逻辑已完成：默认玩具盒研发、制作队列、同步倒计时、加入队列、制作时间展示均可用；但布局和视觉未达到需求图。
- 关卡准备逻辑已完成基础版：选择图纸 / 玩具盒、上限、取消选择、预览内容、开始战斗均可用；但表现形式、库存数量和同类多选逻辑不足。
- 战斗流程已完成基础版：准备阶段、缓冲阶段、波次数、波次倒计时、胜利后返回选关均有逻辑；但 UI 表现仍是工程原型。
- 战斗核心闭环已完成基础版：刷怪、最近友方索敌、守护城堡、建造格、图纸权限、零件消耗、开盒得零件、金币购买玩具盒均已接入。

### 优先处理建议

- P0：先完成关卡挑战链路的 HTML 原型改造，覆盖原始需求里最明显的选关和准备窗口缺口。
- P1：补齐战前配置的玩具盒库存 / 同类多选逻辑。
- P1：统一 `TTDUIStyle`，减少 Widget 内硬编码样式。
- P2：研究所、仓库、战斗 HUD 再按同一玩具桌视觉语言逐步正式化。

## 参考范围

HTML 原型覆盖的核心界面是：

- 顶部导航条：返回按钮、游戏标题、章节标签、玩具盒总数。
- 左侧章节地图：玩具桌背景、路径、玩具装饰、关卡节点、锁定节点。
- 右侧信息面板：`关卡侦察` 和 `战前配置` 两种状态。
- 战前配置：已选图纸槽位、图纸库分类、已选玩具盒槽位、玩具盒库存、最低要求、开始战斗按钮。

默认不采用 WebBrowser 或嵌入 HTML。实现应沿用当前项目的 C++ UMG Widget 和 `TTDUIStyle`。

## P0 - 选关界面改造

- 改造 `UTTDBattleLevelSelectWidget`，从当前纵向关卡列表改为左右布局：
  - 外层为暖色玩具桌背景。
  - 顶部固定导航条，包含返回主界面、标题、章节信息、玩具盒总数。
  - 左侧为章节地图区域。
  - 右侧为关卡信息面板。
- 左侧章节地图需要包含：
  - “当前章节选关”标签。
  - 关卡路径视觉。
  - 若干玩具桌装饰占位。
  - 可点击关卡节点。
  - 锁定关卡节点置灰且不可点击。
- 点击可用关卡节点时：
  - 更新当前选中关卡。
  - 刷新右侧 `关卡侦察` 面板。
  - 保持在选关界面，不立即进入战斗。
- 右侧 `关卡侦察` 面板需要显示：
  - 关卡名。
  - 章节 / 地图标签。
  - 玩具桌预览区域。
  - 关卡信息卡片：目标、难度、地图、怪物。
  - 奖励预览。
  - `准备出战` 主按钮。

## P0 - 战前配置面板改造

- 点击 `准备出战` 后，将右侧面板切换为 `战前配置` 状态。
- 可复用 `UTTDBattlePreparationWidget` 的数据逻辑，但视觉和交互应嵌入或对齐选关界面右侧面板，而不是跳到完全不同的页面风格。
- 战前配置面板需要包含：
  - `返回侦察` 按钮。
  - `使用预设` 按钮。
  - 最低要求提示：至少 1 张图纸、1 个玩具盒。
  - 已选图纸槽位，数量来自关卡配置 `MaxSelectedDiagrams`。
  - 图纸库卡片列表。
  - 已选玩具盒槽位，数量来自关卡配置 `MaxSelectedToyBoxes`。
  - 玩具盒库存卡片列表。
  - 底部状态提示和 `开始战斗` 主按钮。
- 图纸选择规则：
  - 点击未选图纸加入已选槽位。
  - 点击已选图纸取消选择。
  - 达到 `MaxSelectedDiagrams` 后禁用继续选择。
- 玩具盒选择规则：
  - 从研究所存档读取库存或已解锁玩具盒作为可选来源。
  - 支持同类玩具盒重复选择，直到库存或 `MaxSelectedToyBoxes` 达到上限。
  - 选择时库存数字减少。
  - 取消或返回侦察时恢复本次临时选择消耗的库存。
- `开始战斗` 按钮规则：
  - 未满足最低要求时禁用。
  - 满足最低要求后调用当前 `StartPreparedBattle(LevelId, Loadout)` 进入战斗。
  - 启动失败时在面板底部显示失败原因。

## P1 - 数据补齐

- 从 `FTTDBattleLevelDefinition` 填充关卡侦察内容：
  - `DisplayName` -> 关卡名。
  - `Description` -> 关卡说明。
  - `WaveIds.Num()` -> 波次数。
  - `StartingToyBoxes` / `StartingParts` / 后续奖励配置 -> 奖励预览。
- 当前缺少正式字段的数据允许先显示占位，但需要在代码注释或后续文档中集中标记：
  - 章节名。
  - 关卡锁定条件。
  - 难度。
  - 地图预览资源。
  - 怪物情报。
  - 奖励预览。
- 如果后续要正式化，应新增 DataTable 字段或独立关卡 UI 展示配置表，不继续在 Widget 中硬编码。

## P2 - 功能需求补齐

- 图鉴详情：
  - 已解锁图纸和玩具盒点击后改为弹窗详情。
  - 图纸弹窗展示名称、说明和组装零件。
  - 玩具盒弹窗展示名称、说明和可开出零件。
  - 玩具盒零件列表支持横向滚动。
- 研究所视觉：
  - 图鉴网格继续保持每行 5 列。
  - 未解锁内容需要有更明显的置灰图标态。
  - 研发页签改为更接近需求图的上方队列、中间预览、右侧列表布局。
- 战斗商店：
  - 将当前购买 / 打开按钮整理成固定商品商店区域。
  - 商品卡显示价格、库存或可购买状态。
  - 任意阶段均可购买，按钮状态随货币和战斗状态刷新。
- 战斗胜利表现：
  - 胜利时显示独立胜利弹窗或覆盖层。
  - 弹窗显示 5 秒返回倒计时。
  - 倒计时结束后返回选关界面。
- 掉落规则：
  - 将敌人固定 `CurrencyDrop` 扩展为概率掉落配置。
  - 自动化测试覆盖不掉落、必掉落和概率配置边界。
- 建筑属性：
  - 如需求仍要求建筑移动速度，给 `FTTDBattleBuildingRuntimeStats` 增加 `MoveSpeed`。
  - 若建筑首版不移动，则在设计文档中明确 `MoveSpeed=0` 为默认占位，避免逻辑误解。

## 默认实现假设

- 使用 C++ UMG 实现，不嵌入 HTML。
- 优先改 `UTTDBattleLevelSelectWidget`、`UTTDBattlePreparationWidget` 和 `TTDUIStyle`。
- 暂不新增正式美术资源，先用颜色、边框、文字、简单形状和布局复刻 HTML 原型。
- 关卡地图节点位置可先用代码内临时布局，后续再迁移到配置。
