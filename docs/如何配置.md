# 如何配置

本文说明当前版本策划需要修改哪些配置，以及每个字段会影响什么运行行为。

## 配置入口

全局配置入口：

- 编辑器：`Project Settings -> Game -> Toy Tower Defense`
- 配置文件：`Config/DefaultGame.ini`
- 对应类：`UTTDDeveloperSettings`

主要字段：

| 字段 | 作用 |
| --- | --- |
| `CraftingQueueMaxSlots` | 研究所制作队列最大槽位 |
| `SaveSlotName` | 研究所存档槽名 |
| `DefaultUnlockedDiagramIds` | 新存档默认解锁图纸 |
| `DefaultUnlockedToyBoxIds` | 新存档默认解锁玩具盒类型 |
| `DefaultUnlockedPartIds` | 新存档默认解锁零件图鉴 |
| `DefaultBattleLevelId` | 默认进入的战斗关卡 ID |
| `bShowBattleTestCheatButton` | 是否显示战斗测试补给按钮 |
| `GameplayTagDataTables` | `GameplayTag -> DataTable` 映射 |

## DataTable 映射

当前需要绑定这些 DataTable：

| GameplayTag | Row Struct | 用途 |
| --- | --- | --- |
| `TTD.DataTable.Research.Parts` | `FTTDPartDefinition` | 研究所零件 |
| `TTD.DataTable.Research.Diagrams` | `FTTDDiagramDefinition` | 研究所图纸 |
| `TTD.DataTable.Research.ToyBoxes` | `FTTDToyBoxDefinition` | 研究所可制作玩具盒 |
| `TTD.DataTable.ObjectPool.Definitions` | `FTTDObjectPoolDefinition` | 对象池 |
| `TTD.DataTable.Battle.Levels` | `FTTDBattleLevelDefinition` | 战斗关卡 |
| `TTD.DataTable.Battle.Waves` | `FTTDWaveDefinition` | 战斗波次 |
| `TTD.DataTable.Battle.Enemies` | `FTTDEnemyDefinition` | 战斗敌人 |
| `TTD.DataTable.Battle.Buildings` | `FTTDBuildingDefinition` | 战斗建筑 |
| `TTD.DataTable.Battle.ToyBoxRewards` | `FTTDToyBoxRewardDefinition` | 战斗内玩具盒和开盒奖励 |
| `TTD.DataTable.Battle.HeightEffects` | `FTTDBattleHeightEffectDefinition` | 高地属性修饰 |

注意：DataTable 的 Row Struct 必须和上表一致，否则运行时会忽略该表或进入配置错误。

## 研究所零件表

DataTable Row Struct：`FTTDPartDefinition`

| 字段 | 说明 |
| --- | --- |
| `PartId` | 零件 ID，其他表引用这个 ID |
| `DisplayName` | 展示名 |
| `Description` | 描述文本 |

配置建议：

- `PartId` 不要改动已上线存档中正在使用的 ID。
- 新增零件后，如需默认解锁图鉴，加入 `DefaultUnlockedPartIds`。
- 如果零件要用于建筑升级，还需要在建筑表 `UpgradeOptions` 中引用。

## 研究所图纸表

DataTable Row Struct：`FTTDDiagramDefinition`

| 字段 | 说明 |
| --- | --- |
| `DiagramId` | 图纸 ID |
| `DisplayName` | 展示名 |
| `Description` | 描述文本 |
| `RequiredPartIds` | 旧兼容字段，主要用于展示 |
| `PartCosts` | 正式研发消耗，填写零件 ID 和数量 |

配置建议：

- 战斗建筑的 `RequiredDiagramId` 必须引用这里的 `DiagramId`。
- 新增默认图纸时，把 `DiagramId` 加入 `DefaultUnlockedDiagramIds`。

## 研究所玩具盒表

DataTable Row Struct：`FTTDToyBoxDefinition`

| 字段 | 说明 |
| --- | --- |
| `ToyBoxId` | 玩具盒 ID |
| `DisplayName` | 展示名 |
| `Description` | 描述文本 |
| `PossiblePartIds` | 图鉴展示用的可能产出零件 |
| `CraftDurationSeconds` | 制作耗时，单位秒 |

运行规则：

- 制作完成并领取后，增加 `ToyBoxId` 对应的持久玩具盒库存。
- 战前配置携带的玩具盒会从这个持久库存扣除。
- 战斗中开盒实际产出不读本表，而读战斗玩具盒奖励表。

## 战斗关卡表

DataTable Row Struct：`FTTDBattleLevelDefinition`

| 字段 | 说明 |
| --- | --- |
| `LevelId` | 关卡 ID |
| `DisplayName` | 展示名 |
| `Description` | 关卡说明 |
| `Background` | 战场背景和边界配置 |
| `CastleClass` | 城堡 Actor 类 |
| `CastleTransform` | 城堡生成位置 |
| `CastleMaxHealth` | 城堡生命值 |
| `WaveIds` | 本关波次 ID 列表 |
| `StartingDiagramIds` | 不经过战前配置时的默认图纸 |
| `StartingToyBoxes` | 不经过战前配置时的默认战斗内玩具盒 |
| `StartingParts` | 开局战斗内零件 |
| `StartingCurrency` | 开局金币 |
| `StartingBuildingCapacity` | 开局可建建筑数量上限；填 0 时按配置中的默认逻辑处理 |
| `BuildingCapacityPurchaseCost` | 购买建筑上限 +1 的价格 |
| `StartingBuildingLevelCap` | 开局建筑等级上限 |
| `BuildingLevelCapPurchaseCost` | 购买等级上限 +1 的价格 |
| `bAllowArbitraryBuildPlacement` | 是否允许点击地面任意位置建造 |
| `ArbitraryBuildMinSpacing` | 任意位置建造时，建筑之间的最小距离 |
| `MaxSelectedDiagrams` | 战前最多携带图纸数 |
| `MaxSelectedToyBoxes` | 战前最多携带玩具盒数 |
| `PreparationDurationSeconds` | 战斗开始前准备倒计时 |
| `VictoryReturnDelaySeconds` | 胜利后返回选关的延迟 |

配置建议：

- `WaveIds` 中的每个 ID 必须存在于战斗波次表。
- 如果启用任意位置建造，建议给 `ArbitraryBuildMinSpacing` 留出建筑半径和操作空间。
- 如果关闭任意位置建造，场景中需要预放置 `ATTDBuildSlotActor`。

## 战斗波次表

DataTable Row Struct：`FTTDWaveDefinition`

| 字段 | 说明 |
| --- | --- |
| `WaveId` | 波次 ID |
| `DelayBeforeWave` | 本波开始前缓冲时间 |
| `WaveDurationSeconds` | 波次持续时间 |
| `EnemyEntries` | 敌人列表 |

`EnemyEntries` 字段：

| 字段 | 说明 |
| --- | --- |
| `EnemyId` | 敌人 ID，引用敌人表 |
| `Count` | 生成数量 |
| `SpawnInterval` | 生成间隔 |
| `SpawnGroup` | 生成点分组，匹配场景中的 `ATTDBattleSpawnPointActor` |

## 战斗敌人表

DataTable Row Struct：`FTTDEnemyDefinition`

| 字段 | 说明 |
| --- | --- |
| `EnemyId` | 敌人 ID |
| `DisplayName` | 展示名 |
| `EnemyClass` | 敌人 Actor 类 |
| `MaxHealth` | 生命值 |
| `MoveSpeed` | 移动速度 |
| `AttackDamage` | 攻击力 |
| `AttackRange` | 攻击距离 |
| `AttackInterval` | 攻击间隔 |
| `ProgressWeight` | 关卡进度权重 |
| `CurrencyDrop` | 掉落金币数量 |
| `CurrencyDropChance` | 掉落金币概率，0 到 1 |

配置建议：

- `ProgressWeight` 会影响进度条，越高代表该敌人占关卡进度越重。
- `EnemyClass` 建议使用实现对象池接口的敌人类，便于复用。

## 战斗建筑表

DataTable Row Struct：`FTTDBuildingDefinition`

| 字段 | 说明 |
| --- | --- |
| `BuildingId` | 建筑 ID |
| `DisplayName` | 展示名 |
| `RequiredDiagramId` | 需要携带的图纸 ID |
| `BuildingClass` | 建筑 Actor 类 |
| `PartCosts` | 建造消耗零件 |
| `AttackMode` | 攻击模式：即时伤害或投射物 |
| `BaseStats` | 基础属性 |
| `ProjectileClass` | 投射物类，仅投射物攻击模式需要 |
| `UpgradeOptions` | 可用升级配置 |

`BaseStats` 字段：

| 字段 | 说明 |
| --- | --- |
| `MaxHealth` | 生命值 |
| `AttackDamage` | 攻击力 |
| `AttackRange` | 攻击距离 |
| `AttackInterval` | 攻击间隔，数值越小攻击越快 |
| `ProjectileSpeed` | 投射物速度 |
| `MoveSpeed` | 移动速度，建筑通常为 0 |

`UpgradeOptions` 字段：

| 字段 | 说明 |
| --- | --- |
| `PartId` | 用哪种零件升级 |
| `PartCost` | 每次升级消耗数量 |
| `Modifiers` | 升级后应用的属性修饰 |

`Modifiers` 字段：

| 字段 | 说明 |
| --- | --- |
| `Attribute` | 影响属性：攻击、范围、间隔、生命、投射物速度、移动速度 |
| `Operation` | `Add` 增加、`Multiply` 乘法、`Override` 覆盖 |
| `Value` | 修饰值 |

配置示例：

- 齿轮升级攻击：`PartId=Gear`，`PartCost=1`，`Attribute=AttackDamage`，`Operation=Add`，`Value=5`。
- 弹簧升级攻速：`PartId=Spring`，`PartCost=1`，`Attribute=AttackInterval`，`Operation=Multiply`，`Value=0.9`。

注意：如果 `UpgradeOptions` 为空，建筑详情面板会显示没有可用升级零件。

## 战斗玩具盒奖励表

DataTable Row Struct：`FTTDToyBoxRewardDefinition`

| 字段 | 说明 |
| --- | --- |
| `ToyBoxId` | 玩具盒 ID，需要和研究所玩具盒 ID 对齐 |
| `DisplayName` | 展示名 |
| `PurchaseCost` | 战斗商店购买价格 |
| `RollCount` | 打开一次玩具盒抽取多少次奖励 |
| `RewardEntries` | 奖励池 |

`RewardEntries` 字段：

| 字段 | 说明 |
| --- | --- |
| `PartId` | 产出的零件 ID |
| `Weight` | 权重 |
| `MinQuantity` | 单次最小数量 |
| `MaxQuantity` | 单次最大数量 |

配置建议：

- `ToyBoxId` 应和研究所玩具盒表保持一致，避免图鉴展示和实际开盒规则割裂。
- 权重为 0 的条目不会被正常抽中。

## 高地效果表

DataTable Row Struct：`FTTDBattleHeightEffectDefinition`

| 字段 | 说明 |
| --- | --- |
| `HeightEffectId` | 高地效果 ID |
| `Modifiers` | 属性修饰列表，格式同建筑升级 |

使用方式：

- 场景中预放置的 `ATTDBuildSlotActor` 可配置 `HeightEffectId`。
- 任意位置建造当前没有自动高地效果，主要用于固定建造位。

## 对象池表

DataTable Row Struct：`FTTDObjectPoolDefinition`

| 字段 | 说明 |
| --- | --- |
| `ObjectClass` | 要池化的 Actor 或 UObject 类 |
| `InitialSize` | 初始预热数量 |
| `MaxSize` | 最大池大小 |
| `ExpandBy` | 扩容步长 |

配置建议：

- 敌人、投射物、可频繁生成的建筑类建议配置对象池。
- 进入对象池的类必须实现 `UTTDPooledObjectInterface`。
- 未配置但实现接口的类会临时创建，不进入池统计。

## 配置后检查

- 修改 DataTable 后，确认 Row Struct 没有选错。
- 确认所有引用 ID 都能在对应表中找到。
- 关卡表的 `WaveIds`、波次表的 `EnemyId`、建筑表的 `RequiredDiagramId`、奖励表的 `PartId` 都需要逐项核对。
- 修改默认解锁内容或存档结构后，建议删除本地旧存档再验证新手流程。
- UE C++ 变更后需重新编译；纯 DataTable 修改通常可直接在编辑器中保存后运行验证。
