# 免编译打开说明

本文说明其他开发者如何在不本地编译 C++ 的情况下，直接双击 `ToyTowerDefense.uproject` 打开项目。

## 适用范围

- 平台：Windows Win64。
- 编辑器：Unreal Engine 5.8。
- 目标：打开 Editor 版本项目，不是打包后的独立游戏。
- 已上传二进制：项目模块和 `GameplayMessageRouter` 插件的 Development Editor DLL。

当前仓库上传的是打开项目所需的运行二进制：

- `Binaries/Win64/ToyTowerDefenseEditor.target`
- `Binaries/Win64/UnrealEditor-ToyTowerDefense.dll`
- `Binaries/Win64/UnrealEditor.modules`
- `Plugins/GameplayMessageRouter/Binaries/Win64/UnrealEditor-GameplayMessageNodes.dll`
- `Plugins/GameplayMessageRouter/Binaries/Win64/UnrealEditor-GameplayMessageRuntime.dll`
- `Plugins/GameplayMessageRouter/Binaries/Win64/UnrealEditor.modules`

没有上传 `.pdb` 调试符号文件，因为它们体积很大，且不是打开项目必须文件。

## 首次使用步骤

1. 安装 Unreal Engine 5.8，建议使用和本项目一致的 Windows 版本。
2. 拉取仓库，确认上面列出的二进制文件存在。
3. 在项目根目录运行一次：

```powershell
powershell -ExecutionPolicy Bypass -File .\Scripts\SetupDirectOpen.ps1
```

如果脚本没有自动找到 UE 5.8，手动传入引擎根目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\Scripts\SetupDirectOpen.ps1 -EngineRoot "C:\Program Files\Epic Games\UE_5.8"
```

如果想脚本配置完成后直接打开项目：

```powershell
powershell -ExecutionPolicy Bypass -File .\Scripts\SetupDirectOpen.ps1 -Open
```

完成后，后续可以直接双击 `ToyTowerDefense.uproject` 打开项目。

## 脚本做了什么

- 读取 `ToyTowerDefense.uproject` 中的 `EngineAssociation`。
- 在当前 Windows 用户注册表中，把该 `EngineAssociation` 指向本机 UE 5.8 路径。
- 检查项目和插件的预编译 Editor 二进制是否存在。
- 检查二进制 `BuildId` 和本机 UE 版本信息是否明显不一致。
- 可选启动 Unreal Editor 打开项目。

脚本不会生成工程文件，也不会调用 UBT 编译。

## 什么时候仍然会要求编译

以下情况 Unreal Editor 仍可能弹出编译提示：

- 本机 UE 版本不是 5.8，或 BuildId 与上传二进制不兼容。
- 拉取代码后有人修改了 C++ 源码，但没有重新上传对应二进制。
- 删除了 `Binaries/` 或插件 `Binaries/` 下的 DLL。
- 启用了新的 C++ 插件，但没有上传该插件的 Editor 二进制。

遇到这些情况，需要由能编译的开发者重新构建并上传新的二进制，或本机安装 C++ 编译环境后自行编译。

## 给维护者的上传规则

- C++ 代码变化后，先关闭 Unreal Editor，再完整构建 `ToyTowerDefenseEditor Win64 Development`。
- 上传 Development Editor 必需二进制，不上传 PDB。
- 如果插件 C++ 代码变化，也要同步上传插件 `Binaries/Win64` 中对应 DLL 和 `.modules`。
- 确认 `Scripts/SetupDirectOpen.ps1` 能通过二进制检查。
