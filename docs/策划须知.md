# 策划须知

本文面向策划同学，说明当前版本已经落地的玩法规则、验收重点和仍需注意的限制。

## 当前玩法闭环

- 玩家流程：开始界面 -> 主界面 -> 研究所 / 仓库 / 关卡挑战 -> 战前配置 -> 战斗。
- 研究所：支持图鉴、图纸研发、玩具盒制作队列、离线进度补算、完成后领取玩具盒库存。
- 仓库：展示零件库存、已解锁图纸、已解锁玩具盒和玩具盒拥有数量。
- 战前配置：选择已解锁图纸和真实拥有的玩具盒；开始战斗后会扣除携带的玩具盒库存。
- 战斗：守护城堡、波次刷怪、敌人掉金币、购买商品、开玩具盒获得零件、建造建筑、升级建筑、出售建筑、胜利后延迟返回选关。

## 研究所规则

- 制作玩具盒完成后，领取结果是增加玩具盒库存，不会直接拆成零件。
- 零件主要通过战斗中打开玩具盒获得。
- 图纸研发会消耗配置的零件，研发成功后解锁对应图纸。
- 新存档默认解锁内容来自 `Project Settings -> Game -> Toy Tower Defense`。
- 旧存档会自动兼容到当前存档版本，并补充玩具盒库存字段。

## 战前配置规则

- 可携带图纸数量由关卡表 `MaxSelectedDiagrams` 控制。
- 可携带玩具盒数量由关卡表 `MaxSelectedToyBoxes` 控制。
- 玩具盒数量来自真实库存；库存不足时不能开始战斗。
- 战斗启动成功后才扣除携带的玩具盒，启动失败不会扣库存。

## 战斗规则

- 建筑需要同时满足图纸已携带、零件足够、建筑数量未达上限。
- 建筑数量上限来自关卡表 `StartingBuildingCapacity`，战斗中可购买“建筑上限 +1”。
- 建筑等级上限来自关卡表 `StartingBuildingLevelCap`，战斗中可购买“等级上限 +1”。
- 建筑升级由建筑表 `UpgradeOptions` 配置：不同零件可以提供不同属性修饰。
- 升级会消耗战斗内零件，达到当前等级上限后不能继续升级。
- 出售建筑只返还建造消耗的零件，不返还升级消耗的零件。
- 默认支持点击场地地面任意位置建造；仍兼容预放置建造位。
- 任意位置建造当前只校验战场范围和与已有建筑的最小间距，尚未做路径阻挡或复杂碰撞规则。
- 战斗中支持 WASD 移动视角，也支持按住鼠标右键拖动视角。

## 商店商品

- 当前玩具盒数量 +1：购买战斗内玩具盒，消耗金币。
- 可搭建建筑数量上限 +1：消耗金币，提高当前战斗建筑容量。
- 等级上限 +1：消耗金币，提高当前战斗建筑等级上限。

## UI 与交互现状

- 当前战斗 HUD 仍是 C++ 文本/按钮原型，不是最终图标化 UI。
- 建筑升级已提供点击按钮方式作为可玩闭环，还不是正式“拖动零件到建筑”的交互。
- 建筑按钮会根据图纸、零件和建筑上限实时置灰，并显示不可用原因。
- 点击已建建筑会显示详情、属性、消耗记录、升级按钮和出售按钮。

## 策划验收重点

- 研究所完成一个玩具盒制作后，仓库中的玩具盒库存数量增加。
- 战前携带玩具盒进入战斗后，持久库存数量减少，战斗内背包获得对应数量。
- 建筑数量达到上限后，建筑按钮不可选；购买建筑上限后可以继续建造。
- 零件不足时，拥有图纸的建筑也应置灰且不能选中。
- 建筑升级会扣对应零件，建筑等级和属性变化正确。
- 达到等级上限后升级失败；购买等级上限后可继续升级。
- 出售建筑返还建造零件，不返还升级零件。
- 点击地面可建造，过近位置会被拒绝。
- WASD 和鼠标右键拖动能移动视角，且不会移出战场边界。
- 最后一波结束且敌人清空后，显示胜利并在配置时间后返回选关。

- 需要在关卡表里确认建筑上限、等级上限、两种上限商品价格和任意建造间距。
- 当前只配置了少量原型关卡、波次、敌人和建筑，后续需要补正式关卡数据。
- 美术图标、正式拖拽 UI、结算 UI 和关卡锁定进度仍是后续工作。
