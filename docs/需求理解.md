## P0 - 当前缺少或明显不符合需求

- 真实玩具盒库存缺失：`UTTDSaveGame` 当前只有 `UnlockedToyBoxIds`，没有“拥有的玩具盒数量”库存；战前配置里的 `BaseToyBoxStock` 是按已解锁类型临时生成，数量取 `MaxSelectedToyBoxes`，不是真实背包数量。研究所“制作玩具盒”当前领取后直接给零件，也没有产出可携带的玩具盒库存。
- 建筑数量上限缺失：需求里的“可搭建建筑数量上限提高 1”和“建筑数量达上限无法选中建筑图标”尚未实现。当前 `UTTDBattleWorldSubsystem` 只维护 `ActiveBuildings`，没有最大建筑数、当前可建数、购买上限提升、HUD 禁用逻辑。
- 商店商品不完整：当前商店只覆盖“购买玩具盒”。需求列出的固定商品还包括“可搭建建筑数量上限提高 1”和“等级加 1”，这两个商品与对应效果都未实现。
- 建筑升级机制缺失：尚未实现“拖动零件至建筑上消耗零件升级建筑”。当前建筑没有等级、升级消耗记录、按零件类型提供不同升级效果、等级上限检查、商店提升等级上限等逻辑。
- 建筑详情和出售缺失：需求里的点击建筑显示详情窗口、出售建筑回收组建零件、升级零件不返还尚未实现。当前建筑 Actor 没有记录建造消耗用于出售返还，也没有详情 UI。
- 任意位置建造不符合需求：需求写的是点击战斗场地任一位置建造；当前只能点击预放置的 `ATTDBuildSlotActor` 建造位。若继续使用固定建造位，需要和需求重新确认；若按需求实现，需要改为地面命中位置建造并处理碰撞/范围/路径限制。
- WASD/鼠标拖动移动视角缺失：当前战斗开始后 `ATTDMenuPlayerController` 设置 `SetIgnoreMoveInput(true)`、只绑定左键建造点击，没有 WASD 轴输入或鼠标拖拽镜头移动。
- 战斗建筑图标置灰逻辑不完整：需求要求“手中零件不能组装建筑时图标置灰”。当前 HUD 只按携带图纸展示可建建筑，选择后由 `TryBuildOnSlot` 返回零件不足；按钮没有随零件库存实时禁用。
- 战斗 UI 缺正式图标/拖拽交互：当前 HUD 是 C++ 文本/按钮原型，不是需求图里的图标式建筑、玩具盒、零件栏，也没有真正的拖拽零件到建筑升级交互。

## P1 - 已做但存在问题或临时实现

- 战前玩具盒选择库存是临时假库存：虽然支持同类重复选择和数量变化，但不是从存档库存扣减；进入战斗也不会从持久库存消耗。
- `UTTDBattlePreparationWidget` 仍保留旧版独立准备页逻辑：当前主链路更多使用 `UTTDBattleLevelSelectWidget` 内嵌右侧面板，旧 Widget 容易和新交互产生重复维护成本。
- 关卡锁定条件缺失：选关界面有锁定节点视觉占位，但缺少配置化解锁条件、通关进度保存和按进度开放关卡。
- 关卡侦察信息仍有占位：难度、地图名、怪物情报、奖励预览等字段部分硬编码或显示“待配置”，还不是正式关卡展示配置。
- 战斗关卡数量不足：当前配置只有 `PrototypeBattle`，需求截图表现为多关卡地图节点；需要至少补多个关卡数据或明确首版只做 1 关。
- 战斗敌人与波次内容单薄：当前只有 `BasicEnemy` 和 `WaveA`，还不足以支撑完整关卡挑战体验。
- 战斗建筑数量单薄：研究图纸已有多种，但战斗建筑表当前只配置 `BasicTower`、`ProjectileTower` 两种。
- 战斗内玩具盒奖励和研究所玩具盒是两套表：研究所表用于制作/图鉴，战斗奖励表用于开盒奖励；目前展示和实际奖励可能不一致，需要统一策划规则或在 UI 明确来源。
- UI 文案和样式仍大量写在 C++ Widget：不利于后续配置、翻译和美术替换；可逐步迁到 StringTable、DataTable 或 Widget Blueprint。

## 验收标准

- 给定研究所完成一个玩具盒制作，领取后仓库或存档中可看到对应玩具盒数量增加，而不是直接只增加零件。
- 给定战前选择携带玩具盒，进入战斗后持久玩具盒库存减少，战斗内背包获得对应携带数量。
- 给定建筑数量达到当前上限，建筑按钮不可选；购买“建筑上限 +1”后可继续建造。
- 给定拥有某建筑图纸但零件不足，战斗 HUD 中该建筑图标置灰且不能选中。
- 给定拖动或选择某零件升级建筑，建筑等级增加且属性按零件效果变化；达到等级上限后不能继续升级。
- 给定点击已建建筑，显示建筑详情，可出售并返还建造消耗零件，不返还升级消耗零件。
- 给定战斗中按 WASD 或鼠标拖拽，镜头能在战场范围内平滑移动。
- 给定最后一波倒计时结束且敌人清空，显示胜利覆盖层并在 5 秒后返回选关。
