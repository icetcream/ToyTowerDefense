# 玩具塔防功能需求整理

## 目标

根据《功能需求(1).docx》和后续战斗玩法讨论，当前阶段重点实现塔防游戏的前置菜单、研究所系统、全局对象池和首版关卡战斗闭环，为后续仓库、关卡选择、成长和正式内容制作提供基础结构。

## 界面与流程

- 开始游戏界面：展示游戏标题和“开始游戏”按钮，点击后进入主界面。
- 主界面：包含“仓库”“关卡挑战”“研究所”入口；点击对应按钮进入对应界面。仓库当前仍是占位入口；关卡挑战会尝试启动 `DefaultBattleLevelId` 对应的默认战斗，启动失败时需要在主菜单显示明确失败原因。
- 研究所界面：左侧提供“返回”“图鉴”“研发”入口，点击后切换到对应页签。

## 图鉴页签

- 图鉴分为“图纸”和“玩具盒”两类。
- 内容按每行 5 列、至少 2 行的网格排列，并支持向下滚动。
- 未解锁内容图标置灰并不可点击。
- 已解锁内容可点击，点击后显示说明窗口。
- 图纸说明窗口展示图纸名称、文字说明、搭建该图纸所需零件。
- 玩具盒说明窗口展示玩具盒名称、可开出零件总览，零件列表可左右滑动。

## 研发页签

- 进入研发页签默认展示玩具盒研发界面。
- 左下角选项可在“玩具盒”和“图纸”研发间切换。
- 玩具盒研发界面上方为制作队列。
- 制作队列展示正在制作的玩具盒和制作完成剩余时间。
- 队列内所有制作项的时间同步减少。
- 右侧展示所有种类玩具盒。
- 点击某个玩具盒后，中间展示该玩具盒内可开出的零件。
- 点击“加入制作队列”时，如果队列未满，则加入上方制作队列。
- “加入制作队列”按钮下方展示制作所需时间。

## 战斗首版闭环

首版战斗参考“守护城堡”的塔防闭环，并吸收用户提出的“类似土豆兄弟的边界刷怪”和“类似植物大战僵尸的波次进度条”体验。

### 关卡目标

- 关卡开始时在地图中间生成一座我方城堡。
- 战斗目标是守护城堡。
- 城堡血量归零时失败。
- 所有波次生成结束且场上敌人清空时胜利。

### 敌人与波次

- 敌人从场景边界或设计指定位置摆放的 `ATTDBattleSpawnPointActor` 生成。
- SpawnPoint 使用 `SpawnGroup` 区分刷怪组；Wave 中的敌人条目可指定 SpawnGroup。
- 每个 Wave 配置敌人 ID、数量、生成间隔和波次延迟。
- 每种 Enemy 配置血量、移速、攻击、攻击范围、攻击间隔、进度权重和死亡货币掉落。
- 敌人生成后寻找最近的我方目标，目前包括城堡和已放置建筑。

进度条按敌人权重计算：

```text
Progress = 1 - (RemainingUnspawnedWeight + AliveEnemyWeight) / TotalEnemyWeight
```

这里的剩余权重包含尚未成功生成的敌人和场上存活敌人。敌人死亡会减少存活权重并增加关卡内货币。

### 建造与格子

- 关卡中可建造位置由场景预设的 `ATTDBuildSlotActor` 承载。
- BuildSlot 包含 SlotId、格子坐标、高度层级、高度效果器 ID 和占用状态。
- 建造时要求格子未占用。
- 建造时要求玩家拥有对应图纸；图纸只作为权限，不消耗。
- 建造时按建筑配置消耗关卡内零件。
- 高度不作为建造限制。

### 高度效果器

高度通过配置化效果器影响建筑运行时属性，而不是写死为“高处增加射程”等固定规则。

- BuildSlot 的 `HeightEffectId` 对应 `TTD.DataTable.Battle.HeightEffects` 中的一行。
- 效果器由多个属性修改器组成。
- 支持属性：攻击力、攻击范围、攻击间隔、最大生命、投射物速度。
- 支持操作：Add、Multiply、Override。
- 未配置高度效果器时，建筑使用基础属性。

### 建筑攻击

- 建筑配置来自 `TTD.DataTable.Battle.Buildings`。
- 建筑可以使用即时伤害模式：按攻击间隔寻找范围内最近敌人并直接造成伤害。
- 建筑可以使用投射物模式：生成投射物，投射物飞向目标并命中造成伤害。
- 敌人、建筑和投射物优先通过全局对象池获取。

### 玩具盒和零件

- 关卡可配置初始图纸、初始玩具盒、初始零件和初始货币。
- 敌人死亡掉落货币。
- 玩家可用货币购买关卡内玩具盒。
- 开启玩具盒会按配置的权重、抽取次数和数量区间获得零件。
- 获得的零件用于搭建建筑。

## 全局对象池需求

- 项目提供 `UTTDObjectPoolWorldSubsystem` 作为每个 `UWorld` 内的全局对象池。
- 业务侧按 Class 获取对象，不按 GameplayTag 获取对象。
- 只有实现 `UTTDPooledObjectInterface` 的 UObject / Actor 可以进入池或被临时创建。
- 池配置来自 `UTTDDeveloperSettings.GameplayTagDataTables` 中的 `TTD.DataTable.ObjectPool.Definitions`。
- 已配置 Class 按 `InitialSize`、`ExpandBy`、`MaxSize` 运行。
- 未配置但实现池化接口的 Class 可以临时创建，不进入池统计。
- 已配置但达到 `MaxSize` 时获取失败。
- 未实现池化接口的 Class 拒绝创建。

## 配置表需求

所有新增玩法配置继续遵守 `UTTDDeveloperSettings + GameplayTag -> DataTable` 规则。

当前新增配置表 tag：

| GameplayTag | 行结构 | 用途 |
| --- | --- | --- |
| `TTD.DataTable.ObjectPool.Definitions` | `FTTDObjectPoolDefinition` | 对象池容量与扩容 |
| `TTD.DataTable.Battle.Levels` | `FTTDBattleLevelDefinition` | 战斗关卡入口和初始资源 |
| `TTD.DataTable.Battle.Waves` | `FTTDWaveDefinition` | 波次和刷怪节奏 |
| `TTD.DataTable.Battle.Enemies` | `FTTDEnemyDefinition` | 敌人属性和掉落 |
| `TTD.DataTable.Battle.Buildings` | `FTTDBuildingDefinition` | 建筑成本、属性和攻击模式 |
| `TTD.DataTable.Battle.ToyBoxRewards` | `FTTDToyBoxRewardDefinition` | 关卡内玩具盒奖励 |
| `TTD.DataTable.Battle.HeightEffects` | `FTTDBattleHeightEffectDefinition` | 高度效果器 |

## P0 稳定性需求

- 默认战斗启动前必须校验配置和场景承载。
- 缺少默认关卡、Battle DataTable、Wave、Enemy、SpawnPoint 或 BuildSlot 时，启动失败并返回失败原因。
- 配置错误时不生成城堡，不清理已有战斗，不进入假运行状态。
- Enemy Class 如果配置，必须是实现池化接口的 `ATTDBattleEnemyActor` 派生类。
- Projectile Class 如果配置，必须是实现池化接口的 `ATTDBattleProjectileActor` 派生类。
- 运行中敌人生成失败时进入 `ConfigurationError`，不扣除 pending 权重，不误判胜利。
- 投射物必须由战斗 Subsystem 统计并在战斗结束时释放。
- 投射物需要绑定 `BattleInstanceId`，避免上一局投射物命中新一局对象。
