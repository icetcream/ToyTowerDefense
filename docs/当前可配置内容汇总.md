# 当前可配置内容汇总

本文面向策划使用，用来快速查看当前项目中已经接入配置系统的内容、配置位置、当前数据行，以及修改时需要注意的规则。

当前配置主要分为四类：

1. 项目全局配置：默认地图、默认 GameMode、默认关卡、默认解锁内容、存档槽、制作队列长度。
2. 研究所配置：零件、图纸、玩具盒制作。
3. 战斗配置：关卡、波次、敌人、可建造塔、战斗玩具盒奖励、高地效果。
4. 对象池配置：敌人、塔、投射物等运行时对象的池化数量。

当前数据：

| Row | ObjectClass | InitialSize | MaxSize | ExpandBy | 预热 |
| --- | --- | ---: | ---: | ---: | --- |
| `Enemy` | `BP_TTD_BattleEnemy` | 0 | 64 | 4 | 是 |
| `BasicTower` | `BP_TTD_BasicTower` | 0 | 16 | 4 | 是 |
| `ProjectileTower` | `BP_TTD_ProjectileTower` | 0 | 16 | 4 | 是 |
| `Projectile` | `BP_TTD_BattleProjectile` | 0 | 32 | 8 | 是 |

对象池规则：

- 业务侧按 Class 获取对象，不按 GameplayTag 获取。
- 进入池的对象必须实现 `UTTDPooledObjectInterface`。
- 未配置但实现池化接口的 Class 可以临时生成，但不进入池统计。
- 未实现池化接口的 Class 会被拒绝。
- 已配置 Class 达到 `MaxSize` 后不会继续创建池对象。

## 7. 配置修改建议

### 7.1 新增一个可研发图纸

需要修改：

1. `DT_ResearchDiagrams_Manual`：新增图纸行。
2. 如果它要能在战斗中建造塔，还要修改 `DT_BattleBuildings_Manual`，新增建筑行并把 `RequiredDiagramId` 指向这个图纸。

### 7.2 新增一个战斗塔

需要修改：

1. `DT_ResearchDiagrams_Manual`：确保有对应图纸。
2. `DT_BattleBuildings_Manual`：新增建筑行。
3. `DT_ObjectPool_Manual`：如果该塔需要池化，配置对应 `ObjectClass`。
4. 如果是投射物塔，还要配置 `ProjectileClass`，并确保投射物类也在对象池表里配置。

### 7.3 新增一个敌人

需要修改：

1. `DT_BattleEnemies_Manual`：新增敌人行。
2. `DT_BattleWaves_Manual`：在波次 `EnemyEntries` 中引用该敌人。
3. `DT_ObjectPool_Manual`：如果使用新敌人类，配置对象池。

### 7.4 新增一个关卡

需要修改：

1. `DT_BattleLevels_Manual`：新增关卡行。
2. `DT_BattleWaves_Manual`：新增或复用波次。
3. 地图中需要至少有一个 `ATTDBattleSpawnPointActor` 和一个 `ATTDBuildSlotActor`。
4. 如果想让默认战斗进入新关卡，修改 `DefaultBattleLevelId`。

### 7.5 新增一个高地效果

需要修改：

1. `DT_BattleHeightEffects_Manual`：新增高地效果行。
2. 地图中建造位的 `HeightEffectId` 填对应 ID。

## 9. 快速检查清单

改配置后建议检查：

1. 所有 ID 是否拼写一致，例如 `RequiredDiagramId` 必须能在研究图纸表中找到。
2. 关卡 `WaveIds` 是否都存在于波次表。
3. 波次 `EnemyId` 是否都存在于敌人表。
4. 波次 `SpawnGroup` 是否能在地图刷怪点中找到。
5. 建筑 `PartCosts` 使用的零件 ID 是否存在于零件表。
6. 玩具盒奖励 `PartId` 是否存在于零件表。
7. 对象池 `ObjectClass` 是否实现池化接口。
8. 战斗关卡里至少有一个刷怪点和一个建造位。
